# KI-Lehrer: 1:1 Session-Themen

> Strukturierte Themenliste für die Zusammenarbeit mit einem KI-Tutor/Lehrer.
> Geordnet nach Priorität — von "Tag 1" bis "fortgeschritten".

---

## Phase 1: KI-Infrastruktur aufsetzen (Sessions 1-3)

1. **Claude Code auf dediziertem Server einrichten** — Installation, API-Key, Workspace-Struktur
2. **Multi-Window-Workflow mit Claude** — mehrere Sessions parallel, Kontexttrennung, wann welches Modell
3. **MCP-Server aufsetzen** — lokale Tools anbinden (Dateisystem, DuckDB, Browser)
4. **Lokale LLMs installieren** — Ollama/LM Studio einrichten, wann lokal vs. API sinnvoll
5. **Git + Claude Code Workflow** — Versionskontrolle für KI-gestützte Entwicklung

## Phase 2: KI-Grundlagen mit Python (Sessions 4-6)

6. **Python + KI-Bibliotheken** — pandas, scikit-learn, numpy im KI-Kontext (Python-Kenntnisse vorhanden, aber KI-Libraries noch nicht)
7. **Erste ML-Pipeline bauen** — mit eigenen Daten, nicht Iris-Dataset — z.B. SPX-Kursdaten
8. **Feature Engineering für Finanzdaten** — was sind gute Features für Optionshandel
9. **Overfitting verstehen und erkennen** — praktisch, nicht theoretisch

## Phase 3: KI auf Trading anwenden (Sessions 7-10)

10. **KI-gestütztes Backtesting** — wo kann ML die bestehende Engine verbessern
11. **Anomalie-Erkennung** — Regime-Shifts erkennen (offenes Problem im Projekt)
12. **Reinforcement Learning Grundlagen** — relevant für Trading-Bots
13. **LLMs für Datenanalyse** — Claude/GPT als Analysetool für Backtest-Ergebnisse

## Phase 4: Automatisierung & Produktion (Sessions 11+)

14. **CI/CD mit KI** — automatisierte Tests, Deployment
15. **Monitoring & Alerts mit KI** — anomale Trades erkennen
16. **RAG aufsetzen** — eigene Wissensdatenbank durchsuchbar machen (Lessons Learned, Dokumentation)
17. **KI-Agenten bauen** — eigene Agenten die das System überwachen

---

## Empfehlung für die erste Session

Thema 1 + 2 kombinieren: Server einrichten und Multi-Window-Workflow. Das gibt sofort eine funktionierende Arbeitsumgebung.
